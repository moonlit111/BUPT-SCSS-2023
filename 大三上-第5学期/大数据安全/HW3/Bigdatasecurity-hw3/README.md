# 大数据安全HW3

这是一个基于 FIDO2 协议的安全用户管理系统。
FIDO协议：为网站添加用户管理功能，支持FIDO2认证。

## 项目概述

### 主要功能
- ✅ 用户注册和登录
- ✅ FIDO2 无密码认证
- ✅ 生物识别支持（指纹、面部识别）
- ✅ 硬件安全密钥支持
- ✅ 安全的会话管理
- ✅ 用户友好的 Web 界面

### 技术栈
- **后端**: Flask (Python)
- **前端**: HTML5, CSS3, JavaScript
- **认证**: FIDO2 (py-fido2)
- **密码安全**: Werkzeug

## 使用流程

### 注册流程
1. 点击"注册新账户"
2. 输入用户名和密码
3. 系统会自动跳转到 FIDO2 设置页面
4. 按照提示使用生物识别或硬件密钥完成注册
5. 注册完成后自动跳转到登录页面

### 登录流程
1. 点击"登录"
2. 输入用户名和密码
3. 系统验证密码后，跳转到 FIDO2 认证页面
4. 使用生物识别或硬件密钥完成认证
5. 认证成功后进入用户仪表板

4. **会话管理**
   - 问题: 缺少认证状态检查

## FIDO2 工作原理

### 注册过程
1. 用户创建账户并设置密码
2. 系统生成 FIDO2 注册选项
3. 用户的设备（生物识别或硬件密钥）生成密钥对
4. 公钥存储在服务器上
5. 私钥保存在用户设备上

### 认证过程
1. 用户输入用户名和密码
2. 系统验证密码并生成认证挑战
3. 用户的设备使用私钥对挑战进行签名
4. 服务器验证签名
5. 认证成功，用户登录

## 安全特性

- 🔐 **无密码认证**: 使用 FIDO2 标准，无需传输密码
- 🛡️ **密钥隐私**: 私钥永远不离开用户设备
- ✅ **强认证**: 支持生物识别和硬件密钥
- 🔒 **会话安全**: 使用 Flask 会话管理
- 🔑 **密码哈希**: 使用 Werkzeug 进行密码哈希处理

## 注意事项

1. **本地开发**: 当前配置使用 `localhost`，生产环境需要配置有效的域名
2. **HTTPS**: 生产环境必须使用 HTTPS
3. **数据存储**: 当前使用内存存储，生产环境应使用数据库
4. **会话密钥**: 每次启动应用时会生成新的会话密钥，生产环境应使用固定密钥

## 故障排除

### 问题: "此浏览器不支持 WebAuthn"
**解决**: 使用支持 WebAuthn 的现代浏览器

### 问题: FIDO2 注册失败
**解决**: 
- 确保设备支持生物识别或已连接硬件密钥
- 检查浏览器控制台的错误信息
- 尝试刷新页面重新开始

### 问题: 登录后无法进入仪表板
**解决**: 
- 清除浏览器 Cookie
- 检查浏览器控制台的错误信息
- 确保会话未过期
